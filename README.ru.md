## ✨ Не набор твиков — а полноценная runtime-система

AutoSystemBoost — это **нативный C-демон** + shell-оркестратор + WebUI, который принимает решения о поведении CPU, GPU, планировщика и термалов каждые **2 секунды** в реальном времени.

```
┌──────────────────────────────────────────────────┐
│  WebUI — переключение профилей, статус           │
├──────────────────────────────────────────────────┤
│  action.sh → Unix-сокет → команды governor       │
├──────────────────────────────────────────────────┤
│  service.sh — загрузочный оркестратор (1150 стр) │
│  runtime/ — reconcile, watchdog, utils           │
├──────────────────────────────────────────────────┤
│  bin/asb — НАТИВНЫЙ C-ДЕМОН (2766 строк)         │
│    ├── FSM: 6 состояний × 3 профиля              │
│    ├── Гистерезис: 2 тика вверх / 5 тиков вниз   │
│    ├── Телеметрия: 25+ метрик в реальном времени │
│    ├── Память: EMA между перезагрузками          │
│    ├── Обучение: почасовые паттерны нагрузки     │
│    └── epoll: 0% CPU в DEEP_IDLE                 │
├──────────────────────────────────────────────────┤
│  sysfs / procfs / cpufreq / WALT / KGSL          │
└──────────────────────────────────────────────────┘
```

## 🧠 FSM — Конечный автомат из 6 состояний

| Состояние | Условие входа | CPU капы (Balanced) | GPU | Опрос |
|:----------|:--------------|:-------------------:|:---:|:-----:|
| 🌙 `DEEP_IDLE` | Экран выключен | только floor | 0% | 5 сек |
| 💤 `LIGHT_IDLE` | Экран включён, низкая активность | 1.19 / 1.88 ГГц | 15% | 2 сек |
| 📱 `MODERATE` | load ≥ 1.5 | динамически | 40% | 2 сек |
| ⚡ `HEAVY` | GPU ≥ 35% или load ≥ 2.0 | 2.4 / 3.3 ГГц | 65% | 2 сек |
| 🎮 `GAMING` | GPU ≥ 65% | 3.3 / 4.0 ГГц | 100% | 2 сек |
| 🛡️ `SUSTAINED` | t° ≥ 65°C или капы недостижимы | 80% диапазона | 80% | 2 сек |

**Переходы:** ⬆️ Вверх: 2 тика (4 сек) · ⬇️ Вниз: 5 тиков (10 сек) · 📴 Экран OFF → `DEEP_IDLE`: мгновенно · 📱 Экран ON → `LIGHT_IDLE`: мгновенно

**`DEEP_IDLE`:** epoll блокируется = **0% CPU**, ~50 КБ RSS. Просыпается только по uevent экрана или каждые 5 сек для проверки термалов.

## 🛡️ Gap-Aware SUSTAINED

Когда термальный стек OxygenOS урезает частоты GAMING на > **1.5 ГГц** дольше **8 секунд**, governor переходит в `SUSTAINED` — реально достижимый уровень производительности, вместо борьбы с термальной стеной.

| Режим | Поведение | Для чего |
|:------|:----------|:---------|
| `burst` | Быстрый retry, `sustained_level=0.85` | Короткие бенчмарки |
| `stable` | Медленный retry, `sustained_level=0.78` | Длинные игры |
| `auto` | Начинает как burst, деградирует до stable при недостижимых капах | **Рекомендуется** |

Auto логирует причину деградации с контекстом: `auto: degraded burst->stable avg_gap=920k sus=5 gaming=1 sus_pct=72`

## 📈 Телеметрия сессий (25+ метрик)

| Метрика | Описание |
|:--------|:---------|
| `ses_gaming` / `ses_sustained` | Количество входов в GAMING / SUSTAINED |
| `ses_thermal` / `ses_unreachable` | Разбивка причин SUSTAINED |
| `ses_t_heavy/gaming/sustained` | Секунды в каждом состоянии |
| `ses_avg_gap_p0` / `ses_max_gap_p0` | Cap gap в кГц (средний / пиковый) |
| `ses_max_temp` | Пиковая температура °C |
| `ses_t2s` / `ses_t2thermal` | Время до первого SUSTAINED / thermal |
| `ses_efficiency` | Оценка качества 0–100 |
| `ses_recovery` | Количество thermal collapse |
| `bat_deep_idle` / `bat_wake_cycles` | Время в idle / циклы пробуждений |
| `hist_*` | **Постоянные средние между перезагрузками** (EMA, 10 сессий) |

### 🧠 Межсессионная память

Статистика сохраняется в `/data/` при каждом выключении экрана. Переживает перезагрузки. Governor стартует со знанием прошлых сессий.

## 🔧 Команды

```bash
asb status                            # JSON-статус
asb profile:performance               # переключить профиль
asb start-session:performance:auto    # атомарно: профиль + режим + сброс
asb reload                            # перечитать конфиг
asb reset-stats                       # сбросить телеметрию
cat /dev/.asb/state                   # снимок состояния
tail -f /dev/.asb/governor.log        # лог governor в реальном времени

# Анализ (на ПК)
python3 tools/asb_analyze.py log.txt
python3 tools/asb_compare_sessions.py log1.txt log2.txt
```

## 📦 Установка

1. Прошить в **Magisk / KernelSU**
2. Выбрать функции при установке (BT, Камера, CPU, VM, Сеть, WiFi, GPS, Ядро, Логи)
3. Перезагрузить → governor запустится автоматически
4. Открыть **WebUI** → выбрать профиль

## 🏁 В заключение

AutoSystemBoost создан для того, чтобы рутованный OnePlus ощущался **осознанным**:

- быстрее, когда нужна скорость
- спокойнее, когда важна экономия
- умнее в тяжёлых сессиях
- лучший звук и камера без дополнительных приложений
