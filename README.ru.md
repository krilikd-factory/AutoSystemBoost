## ✨ Что такое AutoSystemBoost?

**AutoSystemBoost** — это модуль тюнинга для рутированных устройств **OnePlus**, который улучшает именно те вещи, которые реально ощущаются в повседневном использовании:

- ⚡ более быстрая отзывчивость UI
- 🔋 лучшая эффективность в простое и standby
- 🎮 более сильное поведение в тяжёлых foreground-сценариях
- 🧠 более аккуратная работа scheduler и памяти
- 🖥️ встроенный **WebUI** с живым переключением профилей

Вместо одного статичного набора твиков модуль даёт три понятных режима:

- 🔥 **Performance**
- ⚖️ **Balanced**
- 🔋 **Battery**

---

## 🖥️ Встроенный WebUI

AutoSystemBoost включает полноценный **WebUI** с:

- 📱 отображением модели устройства и чипсета
- 🎛️ живым переключением профилей
- ✅ показом активного профиля прямо в карточке модуля
- 🔗 быстрой ссылкой на Telegram
- 🧩 более аккуратной OnePlus-ориентированной версткой

Это важно, потому что модуль больше не ощущается как “прошил и надеешься на лучшее”.  
Теперь ты реально можешь менять поведение телефона под свой сценарий.

## 🎯 Профили

### 🔥 Performance
Подходит для:
- игр
- тяжёлой многозадачности
- более быстрых запусков приложений
- более сильного top-app приоритета
- меньшей scheduler latency

### ⚖️ Balanced
Подходит для:
- повседневного использования
- плавного UI
- разумного нагрева
- стабильной автономности
- лучшего общего баланса

### 🔋 Battery
Подходит для:
- лёгкого использования
- дней с долгим standby
- меньшего числа wakeups
- более спокойного фонового поведения
- более выраженного приоритета на эффективность

## 📊 Сток vs AutoSystemBoost

> Эти значения — **практические ориентиры и целевая логика тюнинга**, а не лабораторные гарантии.  
> Реальный результат зависит от ядра, ROM, приложений, сети, температуры окружающей среды и сценария использования.

### Поведение в сравнении со стоком

| Категория | Стоковое поведение | Цель AutoSystemBoost |
|----------|---------------------|----------------------|
| Отзывчивость UI | базовый уровень | **~8–18% быстрее по ощущениям** |
| Скорость запуска приложений | базовый уровень | **~5–15% быстрее в foreground** |
| Стабильность анимаций под нагрузкой | возможны небольшие просадки | **более стабильное поведение / меньше микролагов** |
| Ночной standby drain | базовый уровень | **~3–10% лучше в Balanced/Battery** |
| Фоновый шум системы | stock reclaim behavior | **меньше фоновой суеты / чище scheduler** |
| Приоритет foreground-задач | стоковый top-app behavior | **сильнее выражен в Performance** |
| Живое управление тюнингом | нет | **есть через WebUI** |

### Поведение профилей относительно стока

| Профиль | Что меняется относительно стока | Лучший сценарий |
|---------|--------------------------------|-----------------|
| 🔥 Performance | **~10–20% более быстрый foreground feel**, более агрессивное поведение, выше расход | игры, тяжёлые задачи, стресс-сессии |
| ⚖️ Balanced | **~5–12% более цельное daily-поведение**, плавнее и стабильнее | каждый день |
| 🔋 Battery | **~5–20% лучше по standby/light-use эффективности**, меньше wakeups, спокойнее фон | поездки, long-standby, лёгкий сценарий |

### Ожидаемое направление по автономности относительно стока

| Сценарий | Сток | Цель AutoSystemBoost |
|---------|------|----------------------|
| Ночной standby | базовый уровень | **~3–10% лучше** в Balanced / Battery |
| Лёгкое дневное использование | базовый уровень | **~4–9% лучше** в Battery |
| Смешанный daily use | базовый уровень | **~0–6% лучше** в Balanced |
| Тяжёлый гейминг | базовый уровень | **примерно так же или хуже** в Performance, и это нормально |

### Ожидаемое направление по нагреву относительно стока

| Сценарий | Сток | Цель AutoSystemBoost |
|---------|------|----------------------|
| Лёгкое использование | обычное | **слегка спокойнее фон в Battery** |
| Смешанный use-case | обычный | **примерно так же или чуть чище в Balanced** |
| Игры / тяжёлая нагрузка | обычный | **выше sustained heat в Performance**, намеренный компромисс |

## ⚙️ Основные зоны тюнинга

AutoSystemBoost меняет несколько важных частей системы:

- 🧠 **WALT / RAVG scheduler**
- 📈 **uclamp** и top-app weighting
- 🖲️ profile-based CPU frequency floors
- 🎮 GPU idle timing
- 💾 VM tuning, dirty writeback и cache pressure
- 🌐 network / queue tuning
- 🧩 выборочную совместимость vendor/system overlays

Цель не в том, чтобы любой ценой выжать красивую цифру в бенчмарке.  
Цель в том, чтобы телефон реально ощущался быстрее, чище и полезнее в обычной жизни.

## 🌡️ Философия производительности

| Профиль | Нагрев | Автономность | Отзывчивость |
|---------|--------|--------------|--------------|
| 🔥 Performance | выше | ниже | **максимальная** |
| ⚖️ Balanced | умеренный | хорошая | **лучший общий баланс** |
| 🔋 Battery | минимальный из трёх при лёгкой нагрузке | лучшая для standby / light use | самая спокойная |

Battery не должен быть “Performance, только бесплатно и без минусов”.  
Физика, к сожалению, всё ещё продолжает существовать.

## 📦 Установка

1. Прошей модуль через **Magisk / KernelSU / совместимую root-среду**
2. Перезагрузи устройство
3. Открой **WebUI**
4. Выбери профиль под свой сценарий

---

## ✅ Рекомендуемое использование

- Используй **Balanced** как основной daily-профиль
- Переключай **Performance** только когда реально нужна максимальная скорость
- Используй **Battery** в поездках, в дни с долгим простоем или при лёгком use-case

---

## ⚠️ Важные замечания

- Требуется root.
- Лучший результат по-прежнему ожидается на **OnePlus 15**.
- Другие OnePlus теперь поддерживаются заметно безопаснее, чем раньше, но не равны OP15 по уровню оптимизации.
- Профиль Performance намеренно увеличивает нагрев и расход энергии.
- Профиль Battery ставит эффективность выше чистой скорости.
