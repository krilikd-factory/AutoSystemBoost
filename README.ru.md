## ✨ Что такое AutoSystemBoost?

**AutoSystemBoost** — это комплексный модуль оптимизации для рутированных устройств **OnePlus**, который улучшает то, что ты реально ощущаешь:

- ⚡ быстрее отзывчивость интерфейса и запуск приложений
- 🎮 умное поведение под нагрузкой с сессионным governor
- 🔋 настоящая экономия батареи — не просто ограниченные частоты
- 🎵 улучшенное качество звука и Bluetooth-кодеки
- 📷 улучшенный пайплайн обработки камеры
- 🌐 настроенный сетевой стек для меньшей задержки
- 🖥️ встроенный **WebUI** для переключения профилей

Три практических режима:

- 🔥 **Performance**
- ⚖️ **Balanced**
- 🔋 **Battery**

## 🎯 Профили

### 🔥 Performance
- Полный потолок CPU/GPU для SD8 Elite
- Агрессивный WALT-планировщик — топ-апп вес 170, ED boost включён
- Минимальные частоты CPU: `2.1 ГГц` (little) / `2.4 ГГц` (big)
- GPU минимум 25%, таймер простоя 48 мс
- Быстрая запись грязных страниц (800 мс)
- WiFi power save выключен, максимальная очередь TX

### ⚖️ Balanced
- CPU caps при простое: `1.19 ГГц` little / `1.88 ГГц` big, при нагрузке — полный потолок
- WALT топ-апп вес 110, умеренный ED boost
- GPU максимум 85%, таймер простоя 64 мс
- Сбалансированная запись (4 с), умеренный swappiness
- WiFi power save auto

### 🔋 Battery
- CPU caps: `729 МГц` little / `1.07 ГГц` big
- Состояние GAMING заблокировано на уровне governor — нет скачков CPU от GPU-активности
- Быстрый deep idle: governor уходит в DEEP_IDLE через 15 с при низкой активности
- GPU максимум 22%, таймер простоя 16 мс
- Высокий swappiness (180), расширенная запись (180 с)
- WiFi power save включён

## 🧠 Нативный адаптивный Governor (V22)

AutoSystemBoost V22 включает **сессионно-осознанный нативный governor** (`bin/asb`), который работает как демон и принимает решения каждые 2 секунды.

### FSM-состояния

| Состояние | Условие входа | Поведение |
|-----------|--------------|-----------|
| `DEEP_IDLE` | Экран выключен | Минимум пробуждений, опрос каждые 5 с |
| `LIGHT_IDLE` | Экран включён, низкая нагрузка | Консервативные ограничения |
| `MODERATE` | CPU нагрузка ≥ 1.5 | Умеренные ограничения |
| `HEAVY` | GPU ≥ 35% или нагрузка ≥ 2.0 | Активные ограничения, boost доступен |
| `GAMING` | GPU ≥ 65% | Полный потолок, msm_perf boost |
| `SUSTAINED` | Температура ≥ 65°C или GAMING caps недостижимы | Стабильный практичный режим |

### Gap-aware SUSTAINED
Если vendor thermal stack срезает CPU caps в GAMING на более чем 1500 МГц в течение 8+ секунд, governor автоматически переходит в SUSTAINED — режим с более консервативными, но реально применяемыми ограничениями.

### Сессионная телеметрия
Governor отслеживает статистику текущей сессии в `asb status`:
- Время в каждом тяжёлом состоянии
- Средний и пиковый cap gap в GAMING
- Количество входов в SUSTAINED (thermal vs недостижимый)
- Пиковая температура

### Адаптивная high-load стратегия
В `config/governor.conf`:
```
highload_mode=burst    # бенчмарки / короткие сессии
highload_mode=stable   # длинные игровые сессии
highload_mode=auto     # умный переход burst→stable
highload_mode=default  # параметры из конфига
```

### Команды управления
```bash
asb status           # полный JSON-статус
asb profile:battery  # переключить профиль
asb reload           # перечитать governor.conf
asb reset-stats      # сбросить счётчики сессии
tail -f /dev/.asb/governor.log
cat /dev/.asb/state
```

## 🎵 Улучшения звука

AutoSystemBoost применяет полный аудио-стек при установке:

### Качество сигнала
| Область | Стандартно | AutoSystemBoost |
|---------|-----------|-----------------|
| Разрядность наушников | 16/24 бит | **32 бит** |
| Разрядность динамика | 16/24 бит | **32 бит** |
| Точность обработки | PCM 32 бит | **PCM Float** |
| Аппаратный компрессор | Включён (сжимает динамику) | **Выключен — шире динамический диапазон** |
| Soft-clipper | Включён (добавляет искажения) | **Выключен — чище пики** |
| IIR EQ полосы | Включены (окрашивают звук) | **Выключены — ровная АЧХ** |
| Голосовой sidetone | Включён | **Выключен** |

### Громкость и выход
| Область | Стандартно | AutoSystemBoost |
|---------|-----------|-----------------|
| Регистр цифровой громкости | 80–87 / 128 | **88 / 128 (+1–2 дБ)** |
| Уровень HPHL / HPHR | Ограниченный по умолчанию | **Максимум (20/20)** |
| Режим CLSH class-H | NORMAL_DSM_OUT | **CLSH_DSM_OUT — лучший КПД** |
| HiFi режим | Отключён на части прошивок | **Включён** |
| Фильтр верхних частот (HPF) | Включён (срезает sub-bass) | **Выключен — полный низ** |
| Фильтр нижних частот (LPF) | Включён | **Выключен — расширенный верх** |
| VBAT защита (ограничивает выход) | Включена | **Выключена** |

### Частота дискретизации и кодеки
| Область | Стандартный предел | AutoSystemBoost |
|---------|-------------------|-----------------|
| Выход на наушники | До 48 кГц | **До 192 кГц (hi-res audio)** |
| Выход динамик / PCM | 32–48 кГц | **До 384 кГц** |
| Диапазон частот кодека | Фиксированные значения | **1–192 000 Гц разблокировано** |
| Диапазон битрейта | До 320–960 кбит/с | **До 18 Мбит/с** |
| Сложность кодека (Opus/FLAC) | 7–9 / 10 | **Максимум (10 / 10)** |
| A2DP Bluetooth частоты | 44100–96000 Гц | **До 192 000 Гц** |

### Bluetooth аудио
| Область | Стандартно | AutoSystemBoost |
|---------|-----------|-----------------|
| LHDC с низкой задержкой (LL) | Не всегда активен | **Включён** |
| AOSP кодек с низкой задержкой | Не всегда активен | **Включён** |
| A2DP offload | По умолчанию | **Включён — меньше нагрузка на CPU** |
| Trace-логирование BT | Включено (тратит RAM) | **Выключено** |
| AAC frame control | Включён (добавляет задержку) | **Выключен — чище AAC** |

### Аудио offload
Все аппаратные пути декодирования включены: AAC, ALAC, APE, FLAC, PCM 16/24, Vorbis, WMA, Opus. Декодирование переносится на DSP, экономя батарею при воспроизведении музыки.

### Белый список приложений
Neutron Player, Spotify, YouTube Music, Tidal, Qobuz, VK Музыка, Deezer, Apple Music, UAPP, jetAudio и другие добавлены в белый список высококачественной обработки звука.

> **Практический результат:** наушники звучат громче и чище, артефакты компрессии снижены, hi-res файлы воспроизводятся на нативной частоте, Bluetooth-наушники получают поток лучшего качества.

## 📷 Улучшения камеры

| HFR основной камеры | По умолчанию | **Включён** — высокочастотная съёмка |
| Быстрый автофокус | По умолчанию | **Включён** — быстрее захват фокуса |

## 📊 Сравнение с Stock

> Практические целевые диапазоны, не лабораторные гарантии.

### Поведение в ежедневном использовании

| Категория | Stock | Цель AutoSystemBoost |
|---------|-------|---------------------|
| Отзывчивость UI | базовый | **~8–18% быстрее воспринимаемая реакция** |
| Скорость запуска приложений | базовый | **~5–15% быстрее** |
| Стабильность анимации | может падать под нагрузкой | **стабильнее / меньше микростаттеров** |
| Автономность в режиме ожидания | базовый | **~3–10% лучше в Balanced/Battery** |
| Выход наушников | консервативный | **громче, шире динамика** |
| Качество BT аудио | стандартное согласование | **максимальные пути качества** |
| Ночная съёмка камерой | стандартный MFNR | **улучшенное шумоподавление** |

### Автономность vs Stock

| Сценарий | Stock | Цель |
|---------|-------|------|
| Ночной режим ожидания | базовый | **~3–10% лучше** в Balanced/Battery |
| Лёгкое использование | базовый | **~4–9% лучше** в Battery |
| Смешанное использование | базовый | **~0–6% лучше** в Balanced |
| Тяжёлый гейминг | базовый | **равно или хуже** в Performance (намеренно) |

## ⚙️ Основные области тюнинга

- 🧠 **WALT / RAVG планировщик** — тюнинг на каждый профиль
- 📈 **uclamp** — зажимы для топ-апп, foreground и background

## 📦 Установка

1. Прошить модуль через **Magisk / KernelSU**
2. Выбрать функции при установке (BT, Камера, CPU, VM, Сеть, WiFi, GPS, Ядро, Лог)
3. Перезагрузить
4. Открыть **WebUI** → выбрать профиль

---

## ✅ Рекомендованное использование

- **Balanced** — основной ежедневный профиль
- **Performance** — только для игр, бенчмарков, тяжёлых задач
- **Battery** — в поездках, в режиме ожидания, при лёгком использовании

---

## ⚠️ Важные замечания

- Требуется root (Magisk или KernelSU)
- Лучший результат на **OnePlus 15**
- Performance профиль намеренно увеличивает потребление и нагрев
- Battery профиль намеренно снижает пиковую производительность ради КПД
- Улучшения аудио и камеры применяются только на OP15 или при включении соответствующих функций
